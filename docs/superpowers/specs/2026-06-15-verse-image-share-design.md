# 이번 주 성경 구절 — "이미지로 공유" 기능 설계

작성일: 2026-06-15

## 목적

기존 [[2026-06-15-verse-social-design.md]]에서 추가된 "공유" 버튼은 텍스트+링크만
공유한다. 참고 이미지(성경 앱의 "이미지 만들기" 기능)처럼, 배경을 골라 그 위에
구절 텍스트를 예쁘게 올린 **세로 카드 이미지**를 만들어 공유/저장할 수 있게 한다.

## 결정 사항 (사용자 확정)

- **진입점**: 기존 `#shareBtn`을 확장. 탭 시 바텀시트로 "텍스트로 공유"(기존 동작) /
  "이미지로 공유"(신규) 중 선택.
- **이미지 비율**: 9:16 (스토리형). 캔버스 해상도 1080×1920.
- **배경**: 그라디언트 스타일 4종(브랜드 블루 / 파스텔 플로럴 / 다크 모던 / 웜 베이지) +
  사용자 본인 사진(스크림 처리). 갤러리에서 가로 스와치로 선택.
- **카드 내용**: 구절 본문 + 출처. "이번주 암송구절" 라벨은 표시하지 않음.
  카드 하단에 작은 "뉴질랜드 빅토리처치" 워터마크 표시.
- **출력**: "공유하기"(Web Share API, 이미지 파일) + "저장"(PNG 다운로드) 버튼.
- **적용 범위**: "이번 주 구절" 활성 카드의 공유 버튼에만 적용. 지난 구절 모달은
  현재 공유 버튼이 없으므로 범위 제외(기존과 동일).
- Firestore/Storage 변경 없음, 새 이미지 자산 추가 없음(배경은 코드로 생성되는
  그라디언트 + 사용자 로컬 사진).

## UI 구성

### 1) 공유 액션 시트 (신규)

`#shareBtn` 클릭 시 화면 하단에서 올라오는 작은 바텀시트:

```text
🔗 텍스트로 공유      구절 내용 + 링크 공유
🖼️ 이미지로 공유      배경 골라 카드 이미지 만들기
```

- "텍스트로 공유" 선택 → 기존 `shareBtn` 클릭 핸들러 로직(navigator.share
  텍스트/링크, 미지원 시 클립보드 복사) 그대로 실행.
- "이미지로 공유" 선택 → 아래 "이미지 생성 모달"을 오픈.
- 시트 바깥 탭 또는 닫기로 dismiss.

### 2) 이미지 생성 모달 (신규)

세로 레이아웃, 9:16 비율 미리보기를 중심으로 구성:

- **상단**: 미리보기 카드 (`<canvas>` 또는 canvas를 그린 `<img>`).
  - 구절 본문(가운데 정렬, 여러 줄)
  - 출처 (본문보다 작게)
  - 카드 맨 아래: "뉴질랜드 빅토리처치" 워터마크 (가장 작고 옅게)
- **중단**: 배경 스타일 가로 스크롤 갤러리. 스와치 5개, 순서:
  1. 브랜드 블루
  2. 파스텔 플로럴
  3. 다크 모던
  4. 웜 베이지
  5. 내 사진 (📷 아이콘) — 탭 시 `<input type="file" accept="image/*">` 오픈
  - 모달을 열면 기본값은 **브랜드 블루**.
  - 스와치 탭 → 선택 표시(테두리 강조) + 캔버스 즉시 재렌더링.
- **하단**: "공유하기" / "저장" 버튼 2개 (가로 배치).

## 이미지 생성 로직 (Canvas)

### 캔버스

- 크기: 1080×1920 px (off-screen `<canvas>`, 화면에는 CSS로 축소 표시).
- 모달 오픈 시, 모달 닫기 전까지 같은 canvas 엘리먼트를 재사용해 다시 그림.

### 배경 스타일 정의

| 스와치 | 배경 | 본문/출처 글자색 |
| --- | --- | --- |
| 브랜드 블루 | `linear-gradient(160deg, #21428d 0%, #3a5fb8 55%, #6f8fd6 100%)` | `#ffffff` |
| 파스텔 플로럴 | `linear-gradient(165deg, #f6d9e3 0%, #fbeede 55%, #f3e6d8 100%)` | `#7a4a55` |
| 다크 모던 | `linear-gradient(160deg, #2b2b3a 0%, #4a4a63 100%)` | `#ffffff` |
| 웜 베이지 | `linear-gradient(160deg, #fdf6ec 0%, #f1e3c8 100%)` | `#8a6d3b` |
| 내 사진 | 업로드 이미지를 캔버스 전체에 cover-fit으로 그린 뒤, 하단 50% 영역에 `linear-gradient(to top, rgba(20,20,30,.7) 0%, rgba(20,20,30,.05) 100%)` 스크림 | `#ffffff` (고정) |

워터마크 텍스트는 항상 본문 색의 낮은 opacity(예 0.5)로 그림.

그라디언트는 캔버스 좌상단→우하단 대각선 방향(`createLinearGradient(0,0,W,H)`)으로 그린다.

### 텍스트 레이아웃

- 본문(`text`)은 기존 `\n` 줄바꿈을 그대로 줄 단위로 사용(현재 타이핑 애니메이션과
  동일한 줄바꿈을 유지).
- 가운데 정렬, 폰트는 `'Gowun Dodum'`. `document.fonts.ready` 이후에 그려서
  폴백 폰트로 깨지는 것을 방지(기존 `startTypewriting`과 동일 패턴).
- 기본 본문 폰트 크기 64px, 출처 44px, 워터마크 28px.
- 좌우 안전 영역: 캔버스 폭 1080px 기준 좌우 80px 패딩 (텍스트 가용 폭 920px).
- 자동 축소: 각 줄을 `measureText`로 측정해 가용 폭을 넘거나, 줄 수가 많아 세로
  영역을 넘으면 본문 폰트 크기를 36px까지 단계적으로 줄여 모두 들어가도록 함.
- 줄 간격은 폰트 크기의 1.6배.
- 출처는 본문 아래 여백을 두고 배치, 워터마크는 캔버스 하단에서 고정 위치(예: 하단
  80px 지점, 가운데 정렬).

### 갱신 시점

- 모달 오픈 시 기본 스타일(브랜드 블루)로 1회 렌더.
- 스와치 클릭 시 동일 canvas를 다시 그려 미리보기 갱신.
- "내 사진" 선택 후 파일을 고르면, 이미지를 로드한 뒤 해당 배경으로 재렌더링하고
  스와치를 활성 상태로 표시.

## 공유 / 저장 동작

- `canvas.toBlob('image/png')` → `File` 객체 생성 (`victory-verse.png`).
- **공유하기**:
  - `navigator.canShare && navigator.canShare({ files: [file] })`가 true면
    `navigator.share({ files: [file], title: '빅토리처치 이번 주 성경 구절', text: 본문+출처 })` 호출.
  - 지원하지 않는 환경(주로 데스크톱)에서는 "공유하기" 버튼을 표시하지 않고
    "저장" 버튼만 노출.
- **저장**:
  - 항상 동작. `URL.createObjectURL(blob)` + 동적 `<a download="victory-verse.png">`
    클릭으로 PNG 다운로드. 사용 후 `URL.revokeObjectURL`로 정리.

## 파일 변경 범위

- `index.html` 한 곳만 변경 (CSS + 마크업 + JS).
  - 신규: 공유 액션 시트, 이미지 생성 모달, 배경 스와치 갤러리, canvas 렌더링 함수,
    공유/저장 핸들러.
  - 기존 `shareBtn` 클릭 핸들러는 액션 시트를 여는 역할로 변경되고, "텍스트로 공유"
    선택 시 기존 로직을 그대로 호출.
- Firestore 규칙/문서 구조, `firebase-config.js`, `admin.html` 변경 없음.
- 새 이미지 자산 추가 없음.

## 테스트 계획

- 로컬: `npx serve .` (또는 `firebase emulators:start --only hosting`)로 미리보기.
- 확인 항목:
  1. 공유 버튼 → 액션 시트 → "이미지로 공유" → 모달 오픈, 기본 브랜드 블루 카드 렌더링.
  2. 4가지 그라디언트 스와치 전환 시 미리보기가 올바른 배경/글자색으로 갱신되는지.
  3. "내 사진" 선택 → 파일 선택 다이얼로그 → 선택한 사진이 cover-fit + 스크림으로
     배경에 적용되는지.
  4. 짧은 구절(1~2줄)과 긴 구절(5줄 이상) 모두에서 텍스트가 카드 안에 잘 들어가는지
     (자동 축소 동작 확인).
  5. 모바일(iOS Safari / Android Chrome)에서 "공유하기" → 네이티브 공유 시트에
     이미지가 첨부되는지.
  6. 데스크톱에서 "저장" → PNG 파일이 정상 다운로드되는지.
  7. "텍스트로 공유" 선택 시 기존 동작(텍스트+링크 공유/클립보드 복사)이 그대로
     동작하는지(회귀 확인).
